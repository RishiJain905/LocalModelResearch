# Just-in-Time (JIT) Components

## 📌 Overview

> Just-in-Time (JIT) Components refers to AI systems that **dynamically generate, compose, and render UI components at runtime** — rather than relying on pre-defined, developer-coded interfaces. The LLM outputs structured UI definitions (React components, HTML, JSON schemas) that are compiled or interpreted on the fly to produce interactive interfaces.

## 🛠️ Open Source Repos

### CopilotKit / AG-UI Protocol
**Repo:** [github.com/copilotkit/copilotkit](https://github.com/CopilotKit/CopilotKit) | **Stars:** 30.5k | **License:** MIT

> *"The Frontend Stack for Agents & Generative UI. React + Angular. Makers of the AG-UI Protocol."*

**AG-UI Adoption:** Google, LangChain, AWS, Microsoft, Mastra, PydanticAI

**Three patterns of Generative UI:**

| Pattern | Control Level | Freedom Level | Specification |
|---------|---------------|---------------|---------------|
| **Static** | High | Low | AG-UI |
| **Declarative** | Shared | Shared | A2UI, Open-JSON-UI |
| **Open-ended** | Low | High | MCP Apps |

**AG-UI Protocol example:**
```typescript
const weatherAgent = {
  name: "get_weather",
  description: "Get current weather information for a location",
  parameters: z.object({
    location: z.string().describe("The city or location"),
  }),
  handler: async ({ location }) => {
    await new Promise((r) => setTimeout(r, 500));
    return getMockWeather(location);
  },
  render: ({ status, args, result }) => {
    if (status === "inProgress" || status === "executing") {
      return <WeatherLoadingState location={args?.location} />;
    }
    if (status === "complete" && result) {
      const data = JSON.parse(result);
      return <WeatherCard {...data} />;
    }
    return <></>;
  }
};
```

### Renderify
**Repo:** [github.com/webllm/renderify](https://github.com/webllm/renderify)

> *"A runtime-first, bundleless dynamic renderer that lets LLMs produce real, interactive UI on the fly."*

**Problem solved:**
| Existing Approach | Limitation |
|------------------|------------|
| v0 / Bolt.new | Requires full build backend |
| Streamlit / Gradio | Python-based, server-rendered |
| MCP UI | Limited to Markdown + fixed component set |
| Anthropic Artifacts | Closed implementation, not embeddable |
| JSON schema renderers | Cannot express arbitrary UI |

**Pipeline:**
```
LLM output (JSX/TSX or structured plan)
  → CodeGen (parse + normalize)
  → Security policy check (before any execution)
  → Runtime execution (Babel transpile + JSPM module resolution)
  → Rendered UI in the browser
```

**Package topology:**
| Package | Responsibility |
|---------|----------------|
| `@renderify/ir` | Runtime IR contracts |
| `@renderify/runtime` | Runtime execution engine + JSPM loader |
| `@renderify/security` | Policy profiles + static checks |
| `@renderify/core` | Legacy orchestration facade |
| `@renderify/llm` | LLM provider package (OpenAI + Anthropic + Google) |
| `@renderify/cli` | CLI + browser playground |

**Security modes:** `strict` | `balanced` | `relaxed`

### A2UI (Google)
**Repo:** [github.com/google/A2UI](https://github.com/google/A2UI) | **License:** Apache 2.0

> *"An open-source project, complete with a format optimized for representing updateable, agent-generated UIs and an initial set of renderers, that allows agents to generate or populate rich user interfaces."*

**Key Philosophy:**
| Principle | Description |
|-----------|-------------|
| Security First | Declarative data format (not executable code) |
| LLM-Friendly | UI as flat list of components with ID references |
| Framework-Agnostic | Works on web, Flutter, React, SwiftUI, etc. |

**Problem they solved:**
> *"We needed a way to transmit UI that is **safe like data, but expressive like code**."*
> — [Google Developers Blog](https://developers.googleblog.com/introducing-a2ui-an-open-project-for-agent-driven-interfaces/)

## ⚠️ Key Challenges

| Challenge | Description | Solutions |
|-----------|-------------|-----------|
| **Security** | Arbitrary code generation in browser | Sandboxed iframes, declarative formats (A2UI), policy profiles (Renderify) |
| **Consistency** | Generated UIs may look different each time | Component registries, design system constraints |
| **Performance** | Generation takes time | Streaming, progressive rendering |
| **Hallucination** | AI generates non-existent components | Component registry allowlist |
| **Cross-platform** | Different frameworks render differently | A2UI's framework-agnostic declarative format |
