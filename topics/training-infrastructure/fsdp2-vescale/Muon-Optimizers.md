# Muon Optimizers

> **📌 Overview**
> **Muon (MomentUm Orthogonalized by Newton-Schulz)** is a matrix-level optimizer for 2D neural network parameters — applying Newton-Schulz orthogonalization to SGD-momentum updates. Key advantages: **~52% training FLOPs vs AdamW**, **50% memory savings** (1 buffer vs 2), and **~0.57× communication overhead** in distributed settings. Used in production at **MoonshotAI (Kimi K2, Moonlight)** and **Tsinghua (GLM-5)**. Enabled at scale via structure-aware sharding (see [RaggedShard](./Structure-Aware-Sharding.md)).

---

## Definitions & Core Concepts

### Muon Definition

> *"Muon (MomentUm Orthogonalized by Newton-Schulz) is an optimizer for 2D neural network parameters. It works by taking the updates generated by SGD-momentum, and then applying a Newton-Schulz (NS) iteration as a post-processing step to each of them before applying them to the parameters."*

**Source:** [kellerjordan.github.io/posts/muon/](https://kellerjordan.github.io/posts/muon/) — December 8, 2024

### Mathematical Formulation

```
Mₜ = μMₜ₋₁ + ∇Lₜ(Wₜ₋₁)         # Momentum accumulation
Oₜ = Newton-Schulz(Mₜ)          # Orthogonalization via NS iteration
Wₜ = Wₜ₋₁ - ηₜOₜ               # Parameter update
```

**Newton-Schulz Iteration:**
```
Xₖ = aXₖ₋₁ + b(Xₖ₋₁Xₖ₋₁ᵀ)Xₖ₋₁ + c(Xₖ₋₁Xₖ₋₁ᵀ)²Xₖ₋₁
```
Where coefficients: a=3.4445, b=-4.7750, c=2.0315

### Core Insight

> *"SGD-momentum and Adam produce updates with very high condition numbers for transformer hidden layers—essentially low-rank matrices where all neuron updates are dominated by a few directions. Orthogonalization effectively scales up 'rare directions' that have small magnitude but are important for learning."*

**Source:** [kellerjordan.github.io/posts/muon/](https://kellerjordan.github.io/posts/muon/)

### Relationship to Shampoo

> *"Without preconditioner accumulation, Shampoo reduces to: W_{t+1} = W_t - η U V^T. Muon with momentum disabled = 'instantaneous' or 'accumulation-free Shampoo'"*

**Source:** [kellerjordan.github.io/posts/muon/](https://kellerjordan.github.io/posts/muon/)

### Matrix-Level vs Element-Wise

> *"Unlike most conventional optimizers that operate element-wise, Muon applies matrix-level transformations to gradients, enforcing orthogonality"*

**Source:** [arXiv:2602.04669v1](https://arxiv.org/html/2602.04669v1)

## Summary

| Aspect | Details |
|--------|---------|
| **Full Name** | Muon — MomentUm Orthogonalized by Newton-Schulz |
| **Type** | Matrix-level optimizer (vs element-wise like AdamW) |
| **Key Operation** | Newton-Schulz orthogonalization of momentum updates |
| **Best For** | 2D hidden layer parameters in transformers |
| **Pairs With** | AdamW for embeddings, classifier heads, biases |
| **Memory Advantage** | 1 momentum buffer vs AdamW's 2 (50% savings) |
| **Communication** | ~0.57× of distributed AdamW |
| **FLOPs Efficiency** | ~52% of AdamW FLOPs for equivalent performance |
| **Scaling Techniques** | Weight decay + Consistent Update RMS |
| **Key Variants** | MuonClip (Kimi K2), AdaMuon (adaptive second-moment) |
| **Major Users** | MoonshotAI (Kimi K2, Moonlight), GLM-5 |
| **Framework Support** | NVIDIA Megatron Core, veScale, FSDP2, NeMo |
