## 1. What is TDP?

### Definition

**TDP (Thermal Design Power)** is the maximum sustained power a processor (CPU/GPU) is expected to generate under typical workloads, which cooling systems must dissipate to maintain optimal operating temperatures.

> *"TDP refers to the maximum amount of heat a GPU is expected to generate under typical workloads, which cooling systems must dissipate to maintain optimal operating temperatures."*
> — [Massed Compute FAQ](https://massedcompute.com/faq-answers/?question=What%20is%20the%20difference%20between%20thermal%20design%20power%20(TDP)%20and%20board%20power%20(BP)%20in%20a%20custom%20data%20center%20GPU%20design?)

### TDP Limitations

> *"TDP is a simplified metric that represents the average power a processor is expected to consume under specific conditions. However, it fails to capture the dynamic nature of modern workloads and the variability of heat dissipation patterns."*
> — [GRC - The Trouble with TDP: Rethinking Data Center Cooling](https://www.grcooling.com/the-trouble-with-tdp-rethinking-data-center-cooling/)

### TDP vs Board Power (BP)

> *"The TDP of an Nvidia GPU is 'the maximum power that a subsystem is allowed to draw for a 'real world' application, and also the maximum amount of heat generated by the component that the cooling system can dissipate under real-world conditions.'"*
> — [Congress.gov - Data Centers and Their Energy Consumption (R48646)](https://www.congress.gov/crs-product/R48646)

## 5. Power Capping Tools

### NVIDIA nvidia-smi

**Setting Power Limit:**
```bash
# Query current power limit
nvidia-smi -q -d POWER

# Set power limit (in Watts)
sudo nvidia-smi -pl <limit> -sc 0

# Set per-GPU power limit
sudo nvidia-smi -i 0 -pl 140
```

**Source:** [NVIDIA Developer Forum - GPU Power Limiting](https://forum.level1techs.com/t/how-to-set-nvidia-gpu-power-limit-nvidia-smi/131467)

**Power Capping on DGX H100/H200:**

> *"The GPU has three sources of power limits: VBIOS (defines maximum TGP), nvidia-smi tool (sets power limit through host), and SMBPBI (sets power limit via out-of-band channel). The GPU Performance Monitoring Unit (PMU) selects the most conservative policy to cap power consumption."*
> — [NVIDIA DGX H100/H200 User Guide - Managing Power Capping](https://docs.nvidia.com/dgx/dgxh100-user-guide/power-capping.html)

**Kubernetes Integration via GPU Operator:**

> *"To optimize power usage, you can enforce GPU power limits via Kubernetes device plugins. Modify the GPU Operator Helm values to enable power management."*
> — [Massed Compute - NVIDIA Power Management in Kubernetes](https://massedcompute.com/faq-answers/?question=How%20do%20I%20configure%20NVIDIA%27s%20Power%20Management%20Tool%20to%20monitor%20power%20consumption%20in%20a%20Kubernetes%20cluster?)

**Reference:** [GitHub Issue - Support to set power limit through the gpu-operator](https://github.com/NVIDIA/gpu-operator/issues/942)

### AMD rocm-smi

**Power Control Functions:**

```c
amdsmi_set_power_cap(processor_handle, sensor_ind, cap);
amdsmi_set_gpu_power_profile(processor_handle, reserved, profile);
```

**Constraints:** `cap` must be between minimum and maximum power cap values.

> *"Not supported on virtual machine guest."*
> — [AMD SMI Documentation - Power Control](https://rocm.docs.amd.com/projects/amdsmi/en/docs-6.4.0/doxygen/docBin/html/group__tagPowerControl.html)

**Known Issue:**
> *"I am unable to lower the power cap on MI300x. We need to lower the power from 750W to 500W. ROCM-SMI version: 3.0.0"*
> — [GitHub Issue - Unable to lower Power Cap on AMD MI300x](https://github.com/ROCm/ROCm/issues/5231)

## 6. MLPerf Performance-Per-Watt Benchmarks

### MLPerf Power Working Group

> *"The Power working group creates power measurement techniques for various MLPerf benchmarks that enable reporting and comparing energy consumption, performance and power of benchmarks run on submission systems."*
> — [MLCommons Power Working Group](https://mlcommons.org/working-groups/benchmarks/power/)

### MLPerf Power Methodology

**Key Measurement Requirements:**

| Requirement | Specification |
|-------------|---------------|
| **Measurement scope** | Node level (all nodes participating in performance runs) |
| **Measurement point** | At PSUs for DC power |
| **Sampling rate** | At least once per second |
| **Minimum samples** | 60 samples per run |
| **Target accuracy** | 5% (using SPEC or revenue grade power meters) |

**Source:** [MLCommons Training Policies - Power Measurement (GitHub)](https://github.com/mlcommons/training_policies/blob/master/power_measurement.adoc)

**Energy Metric Calculation:**

```
Measured energy = Σ (power_reading × duration)
Estimated energy (interconnects) = max_power × duration × provisioning_ratio
Total energy = Σ energy_per_SUT
```

### MLPerf Training v5.0 Results

> *"MLPerf Training v5.0 introduces new Llama 3.1 405B benchmark. Version 5.0 introduces a new large language model pretraining benchmark based on the Llama 3.1 405B generative AI system, which is the largest model to be introduced in the training benchmark suite."*
> — [MLCommons - MLPerf Training v5.0 Results](https://mlcommons.org/2025/06/mlperf-training-v5-0-results/)

## 8. TDP in Cluster Provisioning and Cooling

### Rack Power Density Requirements

| Configuration | Power per Rack |
|---------------|----------------|
| Traditional data center | 8–20 kW |
| GPU-dense AI cluster | 30–50 kW |
| **Modern AI rack (8× H100)** | **~10-15 kW** |
| **Modern AI rack (8× B200)** | **~15-20 kW** |
| Immersion cooling capable | 50-100+ kW |

**Source:** [Ramboll - 100+ kW per rack in data centers](https://www.ramboll.com/en-us/insights/decarbonise-for-net-zero/100-kw-per-rack-data-centers-evolution-power-density)

### Cooling Requirements (BTU/h Calculation)

```
Power (Watts) → BTU/h = Watts × 3.412
Cooling Tons = BTU/h ÷ 12,000
```

**Example: 8× H100 at 700W each**
- Total power: 5,600W
- BTU/h: 5,600 × 3.412 = 19,107 BTU/h
- Cooling tons needed: 19,107 ÷ 12,000 = **1.6 tons**

### TDP-Driven Infrastructure Decisions

| TDP Factor | Impact |
|------------|--------|
| **Air cooling limit** | ~10 kW/rack (traditional) |
| **Liquid cooling advantage** | Up to 90% reduction in IT cooling power consumption |
| **PUE factor** | GPU-dense: 1.1-1.3; Traditional: 1.5-2.0 |

> *"When compared with conventional air-cooled data centers, immersion cooling cuts power consumption for IT cooling by up to 90%, decreases CapEx by cutting back on costly air-cooling infrastructure, and drives significant OpEx savings directly to the bottom line."*
> — [GRC](https://www.grcooling.com/)

### GPU Operator Configuration for Power Management

To enable power management via Kubernetes GPU Operator:

```yaml
# Helm values for gpu-operator
devicePlugin:
  config:
    nvidia:
      persistence:
        enabled: true
      # Enable power management
      mig:
        enabled: false
```

Run power limit via nvidia-smi:
```bash
sudo chroot /run/nvidia/driver nvidia-smi -pl ${POWER_LIMIT}
```

**Source:** [GPU Operator GitHub Issue #942](https://github.com/NVIDIA/gpu-operator/issues/942)
