# Embedding-Pruning

## 📌 Overview

**Embedding-pruning** uses embedding space properties — cluster membership, centroid distance, and diversity signals — to identify and remove low-quality or redundant samples from datasets. The core premise: high-quality data clusters well and sits close to cluster centers, while low-quality data appears as outliers or in sparse clusters.

> *"Corrupted code tends to reside in smaller clusters and often be farther from cluster centroids."*
> — [SCIP Paper arXiv:2312.02418](https://arxiv.org/html/2312.02418v1)

---

## SCIP — Synthetic Corruption Informed Pruning

### Method

Uses synthetic corruptions to understand what "low-quality" looks like in embedding space.

### Corruption Effects on Embeddings

| Corruption Type | Change Cluster | Change Centroid Distance |
|----------------|---------------|-------------------------|
| Removing Closed Brackets | 77% | 49% |
| Renaming Variables | 89% | 57% |
| Altered Conditionals | 2% | — |
| Offsetting Array Indices | 23% | — |

> *"Renaming variables: 89% change cluster, 57% change centroid distance."*
> — [SCIP Paper](https://arxiv.org/html/2312.02418v1)

### Pruning Strategies

| Strategy | Method |
|----------|--------|
| **Far from Centroid** | Remove points with largest cosine distance to nearest centroid |
| **Small Clusters** | Remove files belonging to smallest clusters |
| **Combined (α=0.8)** | 80% based on cluster size, 20% based on distance |

### Results

> *"Combined method achieved **28.0% on HumanEval** (+3% over no pruning)."*
> — [SCIP Paper](https://arxiv.org/html/2312.02418v1)

## D² Pruning — ICLR 2024

**Diversity & Difficulty** coreset selection via message passing.

### Core Problem

> *"Optimizing for data diversity leads to a coreset biased towards easier samples, whereas selection by difficulty ranking omits easy samples necessary for training."*
> — [D² Pruning ICLR 2024](https://openreview.net/forum?id=thbtoAkCe9)

### Algorithm

1. Build k-NN graph in embedding space
2. Forward message passing — each node receives weighted features from neighbors
3. Reverse message passing — select highest-ranking samples, down-weight neighbors

### Results on ImageNet

| Pruning Rate | D² Pruning | Previous Best |
|-------------|-----------|---------------|
| 50% | **71.8%** | 70.8% (Forgetting) |
| 80% | **65.9%** | 62.9% (Forgetting) |

## SIEVE — Multimodal Dataset Pruning (CVPR 2024)

### Method

> *"Employs synthetic captions generated by image-captioning models pretrained on small, diverse, and well-aligned image-text pairs to evaluate the alignment of noisy image-text pairs."*
> — [SIEVE arXiv:2310.02110](https://arxiv.org/html/2310.02110v1)

1. Generate captions via nucleus sampling using BLIP
2. Mask medium phrases ("image of", "photo of")
3. Compute semantic similarity in sentence transformer embedding space

### Results on DataComp (38 tasks)

| Scale | Method | Avg Performance |
|-------|--------|----------------|
| Medium | CLIPScore | 32.8 |
| | **SIEVE** | **34.6** |
| | **SIEVE+CLIP** | **35.4** |
| Large | CLIPScore | 52.9 |
| | **SIEVE+CLIP** | **54.6** |

## Density-Based Pruning (DBP)

### Core Formula

```
C_j = d_inter × d_intra
```

| Component | Definition |
|-----------|-----------|
| **d_inter** | Avg cosine distance between cluster centroid and its 20 nearest neighbor centroids |
| **d_intra** | Avg cosine distance between cluster items and centroid |

### Pipeline

```
Raw → SemDeDup → CLIP-score filtering → DBP
```

### Results

> *"166M subset (41.6% compute) achieves 65.46% on ImageNet vs 64.07% for full 440M dataset."*
> — [DBP Paper arXiv:2401.04578](https://arxiv.org/html/2401.04578v1)

## SemDeDup — Semantic Deduplication

> *"Identifies semantic duplicates using embedding similarity. Can remove up to 50% of data with minimal performance loss."*
> — [SemDeDup ICLR 2023](https://openreview.net/pdf?id=IRSesTQUtb)

### Algorithm

1. Generate embeddings (CLIP/BLIP)
2. K-means clustering
3. Compute pairwise cosine similarities within clusters
4. Remove above similarity threshold; keep sample with lowest cosine similarity to others

---

## CLIP-Score Filtering

> *"CLIP-score is the cosine similarity between image and caption embeddings... This score method has been widely used to quantitatively measure image-text alignment for data quality control in data pruning."*
> — [DataComp arXiv](https://arxiv.org/html/2310.02110v1)

> *"CLIP-score is the best-performing baseline algorithm in DataComp."*
> — [DataComp arXiv](https://arxiv.org/html/2310.02110v1)

## Quality Signals in Embedding Space

| Signal | Indicates Low Quality |
|--------|---------------------|
| **Small cluster membership** | Outlier/unusual data |
| **Large distance to centroid** | Poor representation |
| **High embedding variance after corruption** | Syntactic/structural issues |
| **Low similarity to synthetic captions** | Poor alignment |

---

## Typical Pipeline

```
Raw Dataset
    ↓
[Embedding Generation] ← Pretrained model (CLIP, BLIP, Sentence Transformer)
    ↓
[Clustering] ← K-means with cosine similarity
    ↓
[Quality Scoring] ← Cluster size, centroid distance, alignment scores
    ↓
[Pruning] ← Remove bottom X% or based on thresholds
    ↓
High-Quality Filtered Dataset
```
